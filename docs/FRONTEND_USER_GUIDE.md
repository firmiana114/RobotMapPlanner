# RobotMapPlanner 前端界面操作指南

适用版本：RobotMapPlanner 0.1.0

文档日期：2026-07-10

## 1. 文档用途

本文说明如何通过 RobotMapPlanner Web 页面完成以下工作：

1. 导入 PCD 点云并生成二维基础占据地图。
2. 创建 Draft，人工补充或清除障碍，并调整地图边界。
3. 验证并发布新的地图版本。
4. 设置起终点，生成等距 XY 路径点。
5. 查看和重新激活历史版本。

系统只处理静态二维地图和全局路径，不控制机器人，也不包含定位、SLAM、局部避障或动态障碍。

## 2. 打开页面与服务状态

当前 WSL 部署地址：

```text
http://172.23.96.128:28200/
```

页面左下角应显示“服务正常 · x86_64”或对应运行架构。若显示“服务异常”，先刷新页面；仍异常时参见第 10 节。

> WSL 地址可能在 Windows 或 WSL 完全重启后变化。无法访问时，在 Windows PowerShell 执行 `wsl hostname -I`，使用输出中的 WSL IPv4 地址和端口 `28200`。

页面左侧包含四个功能入口：

- **地图导入**：导入 PCD、设置投影及代价参数、选择已有地图。
- **地图编辑**：创建和编辑 Draft、验证、发布。
- **规划验证**：设置起点和终点，生成路径点。
- **版本管理**：查看并激活历史发布版本。

页面顶部显示当前地图、活动版本和操作结果。黄色文字表示成功或进行中，红色文字表示失败。

## 3. 推荐操作流程

首次使用建议依次完成：

```text
选择或导入地图
→ 创建草稿
→ 编辑占据图或边界
→ 验证
→ 发布
→ 设置起点与终点
→ 开始规划
```

当前部署已预导入 `global_map_20260708_124133`。如果只需验证规划，可在“地图列表”选择它，然后直接进入“规划验证”。

## 4. 地图导入

### 4.1 选择输入方式

打开“地图导入”，必须先在步骤 1 中选择以下任一地图来源：

- **导入 PCD**：从 Windows 选择本地 `.pcd` 文件，或填写服务允许访问的服务器路径，然后点击“选择此 PCD”。
- **已导入地图列表**：点击已有地图右侧“选择”，复用该地图保存的源 PCD。

当前 WSL 部署允许读取 `/home/u12297/projects`。因此已有点云可以填写：

```text
/home/u12297/projects/global_map_20260708_124133.pcd
```

选择成功后，步骤 2 的地图名称和参数输入框才会启用。选择已导入地图时，页面会自动回填其名称和原始编译参数；若同时选择上传文件并填写服务器路径，系统优先使用上传文件。

只支持 PCD v0.7 ASCII 或 binary XYZ 数据；不支持 `binary_compressed`。

### 4.2 导入参数

| 参数 | 默认值 | 含义与调整建议 |
|---|---:|---|
| 分辨率 | 0.10 m | 每个栅格边长。越小地图越细，但内存和计算量越大。 |
| 障碍下限 | 0.15 m | 相对估计地面的最低障碍高度。过小可能把地面噪声识别成障碍。 |
| 障碍上限 | 2.00 m | 相对地面的最高障碍高度。高于该值的点不参与障碍投影。 |
| 每格最少点 | 1 | 栅格至少命中多少障碍点才算占据。噪声多时可适当提高。 |
| 硬净空 | 0.25 m | 障碍周围不可进入的强制净空距离。 |
| 膨胀半径 | 0.50 m | 障碍向外生成渐变代价的距离，应不小于硬净空。 |
| 代价衰减 | 5.0 | 代价随障碍距离下降的速度。数值越大，远离障碍后的代价下降越快。 |

首次导入建议保留默认值。修改分辨率会改变栅格尺寸；修改净空、膨胀和衰减会影响路径贴近障碍的程度。

### 4.3 执行导入

1. 在步骤 1 选择 PCD 或已导入地图。
2. 在步骤 2 编辑地图名称和编译参数。
3. 新 PCD 点击“导入并编译”；已导入地图点击“按参数重新编译”。
4. 顶部出现“正在编译地图…”时等待完成，不要重复点击。
5. 成功后，地图自动成为当前选择；地图列表显示名称、栅格宽高和源点数。

首次导入会创建不可修改的基础地图和第一个发布版本。已导入地图的重新编译会保持同一地图 ID 和同一列表项，直接替换基础地图；由于栅格尺寸和语义可能变化，旧版本、Draft 和编辑记录会被清除。需要人工修图时必须重新创建 Draft。

### 4.4 删除地图

每个已导入地图的“选择”按钮后方都有“删除”按钮。点击后浏览器会显示确认弹窗；只有确认后才会永久删除该地图的源文件副本、所有版本、Draft、编辑记录和栅格文件。删除不可撤销，操作前应确认不再需要历史数据。

## 5. 地图编辑

### 5.1 选择地图并创建 Draft

1. 在“地图导入”的地图列表点击目标地图右侧“选择”。
2. 打开“地图编辑”。
3. 点击“创建草稿”。
4. 画布下方出现 Draft ID 和 revision 后再开始编辑。

Draft 的每次修改都会增加 revision。它用于防止旧页面覆盖较新的修改。

> 当前页面不能重新打开一个已经存在的 Draft。编辑期间不要刷新页面或切换地图；否则服务端 Draft 仍可能存在，但当前页面不再关联它，需要重新创建 Draft。

### 5.2 图层说明

右侧图层下拉框可以切换：

| 图层 | 用途 |
|---|---|
| 最终占据 | 基础地图与人工覆盖合并后的实际规划输入。 |
| 基础占据 | PCD 自动生成的不可变地图，用于对照原始识别结果。 |
| 人工覆盖 | 只显示人工编辑层，便于检查哪些位置被改过。 |
| 代价地图 | 显示障碍、硬净空和膨胀代价。 |

主要颜色：

- 深黑蓝：未知区域，不能规划。
- 浅灰白：自由区域。
- 红色：障碍或人工强制障碍。
- 青色：人工强制自由。
- 橙色：硬净空区。
- 黄橙渐变：膨胀代价。
- 青色轮廓：当前地图边界。

### 5.3 占据编辑模式

编辑模式包括：

- **补障碍**：将选中栅格设置为 `FORCE_OCCUPIED`，适合补墙、封闭门口或添加不可通行区域。
- **清除误障碍**：设置为 `FORCE_FREE`，适合清除点云噪声造成的假障碍。
- **恢复原值**：设置为 `INHERIT`，删除人工覆盖并重新使用基础地图值。
- **编辑边界**：拖动凸包边界控制点，调整可规划区域。

基础占据图始终不可修改；所有占据修改都保存在覆盖层中。

### 5.4 画刷工具

1. 选择“补障碍”“清除误障碍”或“恢复原值”。
2. 工具选择“画刷”。
3. 调整“画刷”滑块。
4. 在地图上按住鼠标左键拖动，松开后提交本次修改。

画布保留十字光标，并额外显示跟随指针的圆形画刷轮廓；调整滑块时圆形会实时缩放。画刷值是栅格半径，不是米数。例如分辨率为 0.10 m、画刷为 3 时，半径约为 0.30 m。矩形工具和边界编辑模式不显示圆形画刷轮廓。

### 5.5 矩形工具

1. 选择占据编辑模式。
2. 工具选择“矩形”。
3. 在矩形一角按下鼠标左键并拖到另一角。
4. 松开鼠标后，矩形覆盖的全部栅格一次提交。

大面积操作前建议先缩小矩形范围试画，并通过图层切换确认结果。

### 5.6 编辑边界

1. 编辑模式选择“编辑边界”。
2. 边界控制点以黄色方块显示。
3. 在目标控制点附近按下鼠标并拖动。
4. 松开后提交新边界。

只能拖动现有控制点，当前版本不能新增或删除边界点。边界外会变为未知区域且不可规划。边界修改会触发完整代价地图重编译，耗时通常比普通栅格编辑更长。

### 5.7 撤销、重做与验证

- 点击“撤销”回退最近一次 Draft 操作。
- 点击“重做”恢复刚撤销的操作。
- 点击“验证”检查可通行栅格、边界和连通域。

验证成功时顶部会显示类似：

```text
验证通过：可通行 373856，连通域 102
```

连通域大于 1 不一定表示发布失败，但说明自由空间被分成多个互不连通的区域。应确认主要作业区域是否连通，并通过“规划验证”实际测试常用路线。

### 5.8 发布

1. 完成编辑后先点击“验证”。
2. 检查最终占据和代价地图图层。
3. 点击“发布”。
4. 出现“新版本已发布”后，Draft 被关闭，新版本成为活动版本。

发布版本不可修改。发布失败时旧活动版本保持不变。

## 6. 规划验证

### 6.1 设置起点和终点

1. 确认顶部显示的是正确地图和活动版本。
2. 打开“规划验证”。
3. 点击“点击设置起点”，再在地图上单击起点。
4. 点击“点击设置终点”，再在地图上单击终点。

起点显示为绿色，终点显示为黄色。页面顶部会显示点击位置的世界坐标。

### 6.2 规划参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| 点间距 | 0.50 m | 输出 XY 点位之间的最大目标距离。安全拐点会被保留，因此拐点附近的间距可能更小。 |
| 吸附半径 | 0.50 m | 起终点落在障碍或不可行走栅格时，允许寻找附近自由栅格的最大距离。 |
| 最大通行代价 | 0 | 允许路径进入的最高代价值。`0` 会避开完整膨胀半径；仅在明确需要软代价通行时提高，最大为 252。 |

吸附只处理地图范围内的阻塞点。点击在地图栅格范围外会直接失败；吸附半径内没有自由栅格也会失败。

### 6.3 开始规划与查看结果

点击“开始规划”。成功后：

- 青色线表示规划路径。
- 画布下方显示输出点数、路径长度和规划耗时。
- 实际返回数据包含请求起终点、吸附后的实际起终点、等距 XY 点位、路径长度和累计代价。

默认“最大通行代价”为 `0`，规划路径不会进入黄色、橙色或红色代价区。如果提高该值，路径可能进入对应的膨胀渐变区；仍不允许进入 253 硬净空、254 障碍和 255 未知区。如果路线仍过于贴近障碍，应返回地图导入并用更大的膨胀半径重新导入，或在 Draft 中补充障碍。首版不支持直接绘制软代价区或推荐通道。

## 7. 版本管理与回滚

打开“版本管理”可查看当前地图的所有发布版本：

- “当前”表示正在用于规划的活动版本。
- 点击其他版本右侧“激活”，即可将其重新设为活动版本。
- 激活历史版本不会删除新版本，后续仍可再次切换。

版本切换后，“规划验证”会自动使用新的活动版本。切换前应确认当前 Draft 已发布或不再需要。

## 8. 常用操作示例

### 8.1 清除点云造成的孤立假障碍

1. 选择地图并创建 Draft。
2. 图层切到“基础占据”，确认原始红色障碍位置。
3. 模式选择“清除误障碍”，使用小画刷涂抹。
4. 图层切到“最终占据”和“代价地图”检查结果。
5. 验证、发布，再用路径规划测试。

### 8.2 封闭不允许经过的门口

1. 创建 Draft。
2. 选择“补障碍”和“矩形”。
3. 横跨门口画一个连续矩形，避免留下单栅格缺口。
4. 验证后规划一条原本经过门口的路线，确认路径绕行或返回无路径。
5. 发布新版本。

### 8.3 恢复一次错误的人工编辑

- 尚未继续其他操作：点击“撤销”。
- 已有后续操作但只想恢复某片区域：选择“恢复原值”，重新涂抹该区域。
- 已经发布：进入“版本管理”激活发布前的历史版本，或基于当前活动地图创建新 Draft 修正。

## 9. 常见错误与处理

| 页面提示或错误码 | 原因 | 处理方式 |
|---|---|---|
| `uploaded file must use .pcd extension` | 上传文件不是 `.pcd`。 | 选择正确的 PCD 文件。 |
| `source_path is outside configured import roots` | 服务器路径不在允许目录内。 | 当前部署使用 `/imports/...`。 |
| `source file does not exist` | 容器中找不到文件。 | 检查文件名、大小写和挂载路径。 |
| `binary_compressed is not supported` | PCD 使用不支持的压缩编码。 | 转换为 ASCII 或 binary PCD。 |
| `draft revision is stale` / `VERSION_CONFLICT` | 页面持有的 Draft revision 已过期。 | 停止编辑并刷新；当前 UI 无法重新关联旧 Draft，需要重新创建 Draft。 |
| `START_OUTSIDE` / `GOAL_OUTSIDE` | 起点或终点超出地图栅格范围。 | 在地图画布有效范围内重新点击。 |
| `START_BLOCKED` / `GOAL_BLOCKED` | 吸附半径内没有可通行栅格。 | 选择更开阔位置，或适当增加吸附半径。 |
| `NO_PATH` | 起终点之间没有连通路线。 | 检查边界、障碍和连通域，或更换起终点。 |
| `MAP_NOT_READY` | 地图、版本或栅格不可用。 | 重新选择地图；仍失败时检查服务日志和持久化卷。 |
| 页面无地图或画布空白 | 未选择地图，或静态资源缓存异常。 | 返回“地图导入”点击“选择”，然后刷新页面。 |

## 10. 服务检查与维护

在 Windows PowerShell 检查服务：

```powershell
curl.exe http://172.23.96.128:28200/healthz
wsl -d Ubuntu-24.04 -- docker ps --filter name=rabbitbot-map-planner
wsl -d Ubuntu-24.04 -- docker logs --tail 100 rabbitbot-map-planner
```

重启服务：

```powershell
wsl -d Ubuntu-24.04 -- docker restart rabbitbot-map-planner
```

当前容器设置为 `restart unless-stopped`，Docker 服务启动后会自动恢复。地图数据库和栅格保存在 Docker volume `rabbitbot_map_planner_data` 中；删除容器不会删除该卷，但删除卷会永久删除已导入地图、Draft 和版本。

## 11. 操作注意事项

- 编辑前确认地图名称和活动版本，避免修改错误地图。
- 大面积清障后必须检查是否意外打通墙体或未知区。
- 发布前先验证，并至少测试一条常用路线。
- 不要把未知区通过“清除误障碍”大面积强制设为自由，除非已确认现场可通行。
- 不要在导入、发布或版本激活过程中重复点击按钮。
- 页面显示的路径仅是静态全局规划结果，不代表机器人能够安全执行。
- 重要地图发布后应备份持久化数据卷或导出业务所需的版本数据。
